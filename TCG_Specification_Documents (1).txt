--------------------------------------------------------------------------------
  Layer 1 — Foundation  (모든 SSC의 기반, 반드시 읽어야 함)
--------------------------------------------------------------------------------

1. TCG Storage Architecture Core Specification
   버전 : v2.01
   설명 : 토큰 인코딩, SP/Method/Session 구조, 전체 아키텍처의 뿌리.
          모든 SSC가 이 문서를 참조한다.

2. TCG Storage Interface Interactions Specification (SIIS)
   버전 : v1.11 (2023)
   설명 : NVMe/ATA/SCSI 등 인터페이스와 TCG 명령 간 매핑.
          IF-SEND/IF-RECV, Protocol ID, ComID 처리.
          NVMe 개발이라면 필수.


--------------------------------------------------------------------------------
  Layer 2 — SSC Specifications  (구현 대상에 따라 선택)
--------------------------------------------------------------------------------

3. TCG Storage SSC: Opal
   버전 : v2.01 (현행) / v2.02 / v2.30 (draft)
   특징 : 4 Admin / 8 User / 8 LBA Range / 10 MB DataStore / MBR Shadowing.
          가장 풀(Full) 기능. PC 클라이언트 중심.

4. TCG Storage SSC: Ruby
   버전 : v1.00 (2022)
   특징 : Opal 패밀리의 데이터센터/엔터프라이즈 버전.
          1 Admin / 1 User(+옵션) / 128 KB DataStore.
          PBA(Pre-Boot Authentication) 선택적.
          Block SID + PSID Feature Set 필수.

5. TCG Storage SSC: Opalite
   버전 : v1.00
   특징 : Opal 2.01의 서브셋.
          1 Admin / 2 User / Global Range만 / 128 KB DataStore.

6. TCG Storage SSC: Pyrite
   버전 : v1.00 / v2.00 / v2.01
   특징 : 암호화 없음 (Access Control만 제공).
          v2.00부터 Data Removal 기능 추가.
          Block SID Feature Set 필수.

  NOTE: Ruby는 엔터프라이즈/데이터센터 NVMe SSD용으로 Opal과 프로토콜
        호환이지만, PBA가 선택적이고 구조가 단순화되어 있다.
